  <action>
Create `adapter-service/scripts/smoke-test.mjs` -- a standalone Node.js ESM script that:

1. Creates the `__fixtures__/` directory if it does not exist (use `node:fs/promises` mkdir recursive).

2. Patches the scraper's `fetchText` to intercept and save two HTML responses:
   - The first `ajax.fixturelist` response -> save to `__fixtures__/fixturelist.html`
   - The first `/spiel/` match detail response -> save to `__fixtures__/match-detail.html`

   Since the scraper is a standalone script (not importable), use `child_process.execFile` to run `fetch-week.fussballde.mjs` with these env vars:
   ```
   SCOUTPLAN_FROM_DATE = next Saturday (compute dynamically)
   SCOUTPLAN_TO_DATE = next Sunday (compute dynamically)
   SCOUTPLAN_KREIS_ID = moenchen
   SCOUTPLAN_JUGEND_ID = d-jugend
   FUSSBALLDE_PAGE_CONCURRENCY = 2
   FUSSBALLDE_MATCH_CONCURRENCY = 2
   FUSSBALLDE_SAISON = 2526
   SCOUTPLAN_DEBUG_EXPORTER = true
   ```

3. Parse the stdout JSON output and run assertions:
   - `result.games.length > 0` -- at least one game returned
   - For each game, assert:
     - `game.home` is non-empty string
     - `game.away` is non-empty string
     - `game.date` matches `/^\d{4}-\d{2}-\d{2}$/`
     - `game.time` is non-empty and NOT "10:00" (the default fallback) for at least one game
     - `game.venue` is non-empty and NOT "Sportanlage" (the default fallback) for at least one game
     - `game.kreisId === "moenchen"`
     - `game.jugendId === "d-jugend"`
   - `result.meta.provider === "fussball.de"`
   - `result.meta.season === "2526"`

4. For fixture capture: since the scraper runs as a subprocess and we cannot intercept its fetch calls, use a SEPARATE lightweight approach:
   - After the scraper succeeds, extract the first match URL from the scraper's stderr debug output (lines starting with `[fussballde-export]`)
   - OR: parse the returned games JSON, reconstruct a match detail URL from the first game's `id` field (`https://www.fussball.de/spiel/-/-/spiel/${game.id}`), and fetch that page directly with `node:fetch`
   - Also fetch one fixturelist URL by building it from the env vars (use the pattern from the scraper: `ajax.fixturelist/-/staffel/{staffelId}/datum-von/{from}/datum-bis/{to}/max/500/offset/0` -- but since we don't have the staffelId, just save the stderr debug output which contains the URLs)
   - Save both HTML files to `__fixtures__/`

5. Exit with code 0 on success, code 1 on any assertion failure. Print a summary of pass/fail counts.

Use a 120-second timeout for the subprocess (the scraper makes many HTTP requests).
Do NOT use any test framework -- this is a standalone script with manual assertions and process.exit().
